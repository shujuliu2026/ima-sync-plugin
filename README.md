# IMA Sync · IMA 知识库同步

Obsidian 笔记 **推送到腾讯 IMA 知识库**：增量 hash、附件上传、限流退避、匿名统计（默认开启，可关闭）。  
**不依赖**任何特定 CMS 或网站后台 — 仅直连 IMA OpenAPI。

> 产品身份与品牌文案来自插件目录 `product-manifest.json`（白标 fork 时只改此文件）。  
> 开发者 / monorepo 维护见 [`README.dev.md`](README.dev.md)。

---

## 安装

### 方式 A · 官网下载（推荐）

从发行页下载 `ima-sync-v*.zip`，解压到 Obsidian 库的 `.obsidian/plugins/ima-sync/`（须含 `main.js`、`manifest.json`、`styles.css`）。

### 方式 B · 手动复制 dist

若你已有打包好的 `dist/` 目录，将其内容复制到：

```text
<你的库>/.obsidian/plugins/ima-sync/
```

启用：**设置 → 社区插件 → IMA Sync**。

> 勿使用含 `lib/` 子目录的源码树安装；Obsidian 需要单文件 `main.js` bundle。

---

## 配置（三步）

1. **Connection · 连接**：API URL → **API Key + Client ID** → 知识库 ID → 侧栏刷新测通  
2. **Sync · 同步**：添加同步目录 + 自动间隔（0=关）+ 可暂停自动同步  
3. **Advanced · 高级**（可选）：实验拉取、上传间隔 / 分批暂停 / 429 退避、网络重试、Mock 等  

腾讯 IMA 填 `https://ima.qq.com`，须同时填写 Client ID 与 API Key。

## 功能

| 能力 | 说明 |
|------|------|
| **推送到 IMA** | 主功能：上传笔记与附件，增量 hash 跳过未改内容 |
| **同步当前文档** | 推送当前打开的笔记（不受同步目录限制；**免费版可用**，无需 Pro） |
| **同步指定文件夹** | 选择文件夹或推送当前笔记所在目录 |
| **从 IMA 拉取** | **实验·默认关**；可能覆盖本地正文 |
| **全部同步** | **实验·默认关**：推送 + 拉取 |
| **暂停 / 停止** | 批量同步中可暂停或立即停止 |
| **附件同步** | 解析 `![[…]]` / `![](…)` 本地引用 |
| **冲突处理** | 双方均修改时弹窗或默认策略 |
| **断网重试 / 自动重连** | 推送失败重试；未连接时周期探测 |
| **超量/限频提示** | 今日次数用尽、429 时友好 Notice |

> **实验功能（默认关闭）**：「从 IMA 拉取」「全部同步」需在 **高级 → 启用从云端拉取（实验）** 后才会出现。日常推荐仅用 **推送到 IMA**。

## IMA 限流参考

| 类型 | 说明 |
|------|------|
| **QPS** | 约 5 次/秒；超限返回 429，需等待 1–5 分钟 |
| **日额度** | 实测安全 ≤5000 次/天 |
| **插件默认** | 间隔 500ms；每 80 篇休 30s；429 退避 60s→120s→300s |

## 匿名统计

匿名使用统计**默认开启**（版本、活动天数、成功率、错误类型计数）。**不上传** API Key、笔记路径或正文。  
可在 **反馈与帮助改进** 中关闭；关闭后除下方例外外，新数据仅保留在本机。

**例外**：首次启用会上报一次 `ima.install`（插件版本与大致地域，用于安装量统计），与是否开启匿名统计无关。
